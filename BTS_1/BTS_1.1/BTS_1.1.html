Esercizio BTS_1.1 - Bootstrap Basics

Domanda 1: A cosa serve Bootstrap?

Bootstrap è un framework di sviluppo front-end open-source che serve a creare siti web e applicazioni web responsive e mobile-first.

Principali utilizzi:

  • Semplifica la creazione di interfacce web moderne e professionali
  • Fornisce componenti CSS e JavaScript predefiniti (navbar, moduli, pulsanti, card, etc.)
  • Include un potente sistema a griglia per creare layout responsive
  • Garantisce compatibilità cross-browser
  • Riduce i tempi di sviluppo grazie a classi riutilizzabili

Domanda 2: Cosa devi importare per creare una pagina web con Bootstrap?

Per utilizzare Bootstrap 5 in una pagina web, devi importare:

1. File CSS di Bootstrap (nella <head>):
<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.0.2/dist/css/bootstrap.min.css" rel="stylesheet">
2. File JavaScript di Bootstrap (prima della chiusura </body>):
<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.0.2/dist/js/bootstrap.bundle.min.js"></script>
3. Meta tag viewport (nella <head>):
<meta name="viewport" content="width=device-width, initial-scale=1">
Nota: Bootstrap 5 non richiede più jQuery, a differenza delle versioni precedenti.

Domanda 3: Cos'è un CDN? Che vantaggi comporta?

CDN (Content Delivery Network) è una rete di server distribuiti geograficamente in tutto il mondo che ospitano e distribuiscono contenuti statici come librerie CSS e JavaScript.

Vantaggi di usare un CDN:
  • Velocità di caricamento: i file vengono serviti dal server più vicino alla posizione geografica dell'utente, riducendo la latenza
  • Caching del browser: se l'utente ha già visitato altri siti che usano lo stesso CDN, i file saranno già in cache e il caricamento sarà istantaneo
  • Riduzione del carico sul server: i file statici vengono serviti dal CDN, alleggerendo il tuo server
  • Affidabilità: se un server CDN non è disponibile, il traffico viene automaticamente reindirizzato ad altri server
  • Larghezza di banda: risparmio sui costi di bandwidth del tuo hosting
  • Sicurezza: protezione contro attacchi DDoS
Svantaggio: Richiede una connessione internet attiva. Per lavorare offline è necessario scaricare Bootstrap localmente.

Domanda 4: A cosa serve il meta tag viewport?

<meta name="viewport" content="width=device-width, initial-scale=1">

Questo meta tag è fondamentale per il responsive design e serve a:

  • width=device-width: imposta la larghezza della viewport uguale alla larghezza dello schermo del dispositivo (320px su smartphone, 768px su tablet, ecc.)
  • initial-scale=1: imposta il livello di zoom iniziale al 100% quando la pagina viene caricata per la prima volta
Perché è importante?

Senza questo tag, i browser mobili renderizzerebbero la pagina con una larghezza desktop (tipicamente 980px) e la ridimensionerebbero per farla entrare nello schermo, rendendo il testo molto piccolo e difficile da leggere.

Con il meta tag viewport, diamo al browser istruzioni precise su come gestire le dimensioni della pagina, permettendo al nostro layout responsive di funzionare correttamente su tutti i dispositivi.

Risultato: La pagina si adatta perfettamente allo schermo del dispositivo senza necessità di zoom o scroll orizzontale.

Domanda 5: Differenza tra "container" e "container-fluid"

Container (.container)
  • Ha una larghezza massima fissa che varia in base ai breakpoint
  • È centrato nella pagina con margini laterali
  • La larghezza si adatta ai breakpoint:
    • < 576px: 100% width
    • ≥ 576px (sm): max-width 540px
    • ≥ 768px (md): max-width 720px
    • ≥ 992px (lg): max-width 960px
    • ≥ 1200px (xl): max-width 1140px
    • ≥ 1400px (xxl): max-width 1320px
  • Ideale per: contenuti centrali, layout standard con larghezza controllata
Container-Fluid (.container-fluid)
  • Ha sempre larghezza 100% del viewport
  • Si estende da bordo a bordo dello schermo
  • Non ha margini laterali
  • Ideale per: header, footer, sezioni a tutta larghezza, gallerie fotografiche
Consiglio: Spesso si combinano i due: container-fluid per header/footer e container per il contenuto principale.
